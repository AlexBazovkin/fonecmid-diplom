**Установка значений констант:**

---
**Важно!**

Для работы системы учета необходимо, перед установкой констант, создать элементы «Абонентская плата» и «Работы специалиста» (вид номенклатуры – «Услуга») в справочнике «Номенклатура». Данные элементы справочника понадобятся в дальнейшем для заполнения значений констант.

---



**Примечание:** Для удобства установки значений констант воспользуйтесь формой «Настройки расширения
(константы)» в подсистеме «Администрирование».

1. Токен управления телеграмм ботом – уникальный код, по которому
бот идентифицируется в Telegram.
2. Идентификатор группы для оповещения – это идентификатор, который
используется для настройки отправки уведомлений в групповой чат.
3. Таймаут соединения – определяет время ожидания осуществляемого
соединения и операций, в секундах (по умолчанию – 0 – таймаут не
установлен.

**Примечание:** Для проверки корректности настройки параметров телеграмм бота
пользователь с правами администратора может воспользоваться обработкой
«Отправка сообщений в телеграм» в подсистеме
«Обслуживание клиентов».

4. Номенклатура абонентская плата – элемент справочника
Номенклатура, соответствующей абонентской плате.
5. Номенклатура работы специалиста – элемент справочника
Номенклатура, соответствующей выполненным в рамках абонентского
обслуживания клиента работам специалиста.
6. НДФЛ – величина на доходы физических лиц (в процентах).
7. Максимальное число дней отпуска – суммарная величина отпуска в
календарных днях, при превышении которого система будет выдавать
предупреждение (рекомендуемое значение – 28).
